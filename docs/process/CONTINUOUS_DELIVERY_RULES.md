# 持续交付规则

本规则用于约束 Agent 在执行需求时不要停在计划、分析或部分完成阶段。

## 总原则

需求一旦被项目经理 Agent 接收，所有被调度的 Agent 都必须持续推进到自己的职责边界完成。

除非出现“必须用户确认的阻塞”，否则 Agent 不应停止在以下状态：

- 只完成计划但未创建交付物。
- 只完成分析但未输出文档。
- 只完成部分页面或部分接口。
- 只说明下一步但没有实际更新文件。
- 可以合理假设继续推进，却等待用户确认。

如果信息不足但可以合理假设，Agent 应先继续推进，并把假设写入对应文档。

## 项目经理 Agent 的最终目标

项目经理 Agent 是唯一对“整体需求完成”负责的角色。

项目经理 Agent 的最终目标不是创建任务，而是持续调度直到：

1. 产品经理 Agent 已完成需求解析和验收标准。
2. 后端 Agent 已完成需求范围内的接口、数据、权限、校验和自测。
3. 前端 Agent 已完成需求范围内的全部目标页面、组件、交互、状态和联调。
4. 测试 Agent 已完成测试计划、执行验证和 QA 报告。
5. 没有未关闭的 P0/P1 缺陷。
6. 用户目标中的全部页面和关键流程已经完成。

只有满足以上条件，项目经理 Agent 才能把需求标记为 `done`。

## 单个 Agent 的最终目标

单个 Agent 不以“全部页面完成”作为自己的最终目标。单个 Agent 的最终目标是完成自己职责内的任务，并把结果写入文档，等待项目经理 Agent 的下一步调度。

### 产品经理 Agent

最终目标：

- 输出 PRD。
- 输出页面范围、用户故事、业务规则和验收标准。
- 标记假设和未决问题。
- 将结果写入 `docs/requirements/`。
- 完成后等待项目经理 Agent 调度。

### 后端 Agent

最终目标：

- 完成需求范围内的 Spring Boot 后端开发。
- 完成 MyBatis-Plus 数据访问逻辑。
- 完成 MySQL 表结构、字段、索引或迁移说明。
- 完成接口契约。
- 完成后端自测或测试说明。
- 将结果写入 `docs/api/`、`docs/tasks/` 或相关实现说明。
- 完成后等待项目经理 Agent 调度。

### 前端 Agent

最终目标：

- 完成任务范围内的 Vue 页面、组件、路由、状态、表单、异常状态和接口联调。
- 完成前端自测或测试说明。
- 将结果写入 `docs/tasks/` 或相关实现说明。
- 完成后等待项目经理 Agent 调度。

### 测试 Agent

最终目标：

- 输出测试计划。
- 执行验证。
- 输出 QA 报告。
- 明确验收结论：`pass`、`pass_with_risk`、`fail` 或 `blocked`。
- 将结果写入 `docs/testing/` 和 `docs/reports/`。
- 完成后等待项目经理 Agent 调度。

## 必须用户确认的阻塞

只有以下情况可以暂停并请求用户确认：

1. 需求存在冲突，并且不同选择会导致明显不同的产品行为。
2. 缺少数据库连接、账号、密钥、原型访问权限、第三方服务配置等必要外部信息，导致无法继续实现或验证。
3. 涉及破坏性数据库变更、数据删除、权限放开、安全风险或不可逆操作。
4. 现有技术栈、目录结构或部署方式与任务要求冲突，无法安全判断采用哪种方案。
5. 需求涉及支付、短信、实名认证、登录授权等外部服务，但缺少供应商、账号或配置。
6. 用户明确要求暂停、确认或等待。

## 非阻塞情况

以下情况不能作为停止理由：

- 文案细节可以先按常规后台系统写。
- UI 细节可以先遵循现有组件库或原型。
- 字段类型可以基于 MySQL 常规类型先设计，并在文档中标记假设。
- 接口路径可以按 RESTful 风格先设计，并在 API 契约中记录。
- 缺少最终视觉稿但已有原型或需求说明。
- 测试数据可以用本地 mock 或种子数据构造。

## 阻塞输出格式

如果确实必须暂停，Agent 必须输出：

```markdown
## Blocked

- Requirement ID:
- Task ID:
- Current Agent:
- Blocking Reason:
- Why User Confirmation Is Required:
- Options:
- Recommended Option:
- Minimal Question:
- Files Updated Before Blocking:
```
