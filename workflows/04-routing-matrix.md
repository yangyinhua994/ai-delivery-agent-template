# 调度矩阵

项目经理 Agent 使用此矩阵判断应该调度哪个 Agent。

| 输入类型 | 识别特征 | 首个 Agent | 后续 Agent |
| --- | --- | --- | --- |
| 原型地址 | 包含 URL、页面设计、交互原型 | 产品经理 | 后端、前端、测试 |
| 新功能 | “新增”“实现”“做一个” | 产品经理 | 后端、前端、测试 |
| 需求变更 | “改成”“新增字段”“调整流程” | 产品经理 | 视影响范围决定 |
| 前端缺陷 | 样式、布局、交互、页面状态、按钮、表单展示 | 前端 | 测试 |
| 后端缺陷 | 接口、数据、权限、校验、性能、错误码 | 后端 | 测试 |
| 联调问题 | 前端表现和接口返回不一致 | 项目经理判断 | 前端、后端、测试 |
| 回归验证 | “已修复”“帮我验收”“重新测试” | 测试 | 项目经理 |
| 需求不清 | 范围、规则、验收标准不明确 | 产品经理 | 项目经理 |

## 归属判断细则

前端 Agent 负责：

- 页面布局
- 组件状态
- 表单交互
- 路由跳转
- 浏览器兼容
- 接口调用适配

后端 Agent 负责：

- 接口实现
- 数据模型
- 权限校验
- 业务规则
- 错误码
- 性能和安全

产品经理 Agent 负责：

- 页面范围
- 流程规则
- 字段含义
- 用户故事
- 验收标准
- 需求变更影响分析

测试 Agent 负责：

- 测试计划
- 测试用例
- 回归测试
- 缺陷报告
- 验收结论
