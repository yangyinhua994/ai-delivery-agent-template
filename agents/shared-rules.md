# 共享规则

所有 Agent 都必须遵守这些规则。

## 工作原则

1. 只做自己角色职责内的事情。
2. 所有结论必须落到文件，不能只停留在聊天记录。
3. 如果输入不完整，先列出阻塞点和最小澄清问题。
4. 如果能基于现有信息合理推进，先推进，再把假设写清楚。
5. 不覆盖其它 Agent 的交付物，除非任务明确要求修订。
6. 必须读取并遵守 `docs/process/CONTINUOUS_DELIVERY_RULES.md`。

## 持续推进规则

除非出现必须用户确认的阻塞，否则 Agent 不应停止在计划、分析或部分完成阶段。

单个 Agent 的目标是完成自己职责范围内的交付物，并写入文档后等待项目经理 Agent 的下一步调度。

项目经理 Agent 的目标是持续调度，直到用户目标中的全部页面、前端、后端、测试和验收都完成。

必须用户确认的阻塞包括：

- 需求冲突导致无法判断产品行为。
- 缺少数据库连接、账号、密钥、原型访问权限或第三方配置，导致无法继续实现或验证。
- 涉及破坏性数据库变更、数据删除、权限放开、安全风险或不可逆操作。
- 技术栈、目录结构或部署方式与任务要求冲突，无法安全判断方案。
- 用户明确要求暂停、确认或等待。

## 默认技术框架

所有 Agent 必须读取并遵守 `docs/architecture/TECH_STACK.md`。

- 前端：Vue，默认 Vue 3。
- 后端：Spring Boot。
- 持久层：MyBatis-Plus。
- 数据库：MySQL。

除非用户或项目经理 Agent 明确变更，否则不得擅自替换技术栈。

## 文件命名

需求编号使用 `REQ-YYYYMMDD-001` 格式。

推荐文件命名：

```text
docs/requirements/REQ-20260616-001-prd.md
docs/tasks/REQ-20260616-001-tasks.md
docs/api/REQ-20260616-001-api-contract.md
docs/testing/REQ-20260616-001-test-plan.md
docs/reports/REQ-20260616-001-qa-report.md
```

## 交付物状态

使用以下状态：

- `draft`：草稿
- `ready`：可执行
- `in_progress`：执行中
- `blocked`：阻塞
- `review`：等待检查
- `done`：完成

## 输出格式

每个 Agent 的输出必须包含：

- 本次任务编号
- 读取的输入文件
- 产出的文件
- 明确假设
- 阻塞问题
- 下一步建议
